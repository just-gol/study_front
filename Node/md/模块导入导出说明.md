# Node 模块导入导出说明

## 1. `import` 和 `require` 的区别

`import` 不是 TypeScript 专用，`require` 也不是 JavaScript 专用。

它们的真正区别是模块规范不同：

- `import / export` 属于 `ES Module`
- `require / module.exports` 属于 `CommonJS`

可以这样记：

- `import` = 现代标准模块语法
- `require` = Node 传统模块语法

## 2. 两种写法的最直观区别

### CommonJS

```js
const fs = require("fs");
```

### ES Module

```js
import fs from "fs";
```

这两句都是在引入 `fs` 模块，只是写法所属的模块系统不同。

## 3. 为什么会觉得 `module.exports` 更简单

因为 `CommonJS` 的思路更像“对象进出”：

- 导出用 `module.exports`
- 导入用 `require`

写起来很直白：

```js
module.exports = {
  a,
  b
};
```

```js
const obj = require("./me.js");
```

而 `ES Module` 需要区分：

- `export`
- `export default`
- `import {}`
- `import xx from`

所以刚学的时候会觉得 `module.exports` 更顺手，这是正常的。

## 4. 导出多个内容的对应关系

### CommonJS 写法

`me.js`

```js
function tiemo() {
  console.log("贴膜");
}

function niejiao() {
  console.log("捏脚");
}

module.exports = {
  tiemo,
  niejiao
};
```

`index.js`

```js
const { tiemo, niejiao } = require("./me.js");

tiemo();
niejiao();
```

### ES Module 写法

`me.js`

```js
function tiemo() {
  console.log("贴膜");
}

function niejiao() {
  console.log("捏脚");
}

export { tiemo, niejiao };
```

`index.js`

```js
import { tiemo, niejiao } from "./me.js";

tiemo();
niejiao();
```

### 对照理解

- `module.exports = { tiemo, niejiao }`
- `export { tiemo, niejiao }`

这两种都表示“导出多个成员”。

## 5. 导出一个主要内容的对应关系

### CommonJS 写法

`me.js`

```js
function tiemo() {
  console.log("贴膜");
}

module.exports = tiemo;
```

`index.js`

```js
const tiemo = require("./me.js");
tiemo();
```

### ES Module 写法

`me.js`

```js
function tiemo() {
  console.log("贴膜");
}

export default tiemo;
```

`index.js`

```js
import tiemo from "./me.js";
tiemo();
```

### 对照理解

- `module.exports = 某个值`
- `export default 某个值`

学习时可以先这样类比。

## 6. `import` 里有没有花括号，区别很大

### 命名导出

如果是：

```js
export { tiemo, niejiao };
```

导入时要写：

```js
import { tiemo, niejiao } from "./me.js";
```

### 默认导出

如果是：

```js
export default tiemo;
```

导入时要写：

```js
import tiemo from "./me.js";
```

### 快速记忆

- 有花括号：拿命名导出
- 没花括号：拿默认导出

## 7. `__filename` 为什么在 `import` 里报错

如果文件是 `ES Module`，下面这种写法会报错：

```js
console.log(__filename);
```

因为：

- `__filename`
- `__dirname`

这两个变量属于 `CommonJS`，不属于 `ES Module`。

在 `ES Module` 里要这样写：

```js
import { fileURLToPath } from "url";
import path from "path";

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
```

## 8. JSON 导入为什么会报错

如果你写：

```js
import text from "./text.json" assert { type: "json" };
```

报错通常不是 JSON 内容错了，而是当前 Node 环境不支持这段语法。

另外要注意，JSON 导入后得到的是对象，不是函数。

例如：

```js
console.log(text);
console.log(text.name);
```

这样是对的。

下面这样通常会报错：

```js
text();
```

因为 `text` 是对象，不是函数。

## 9. 现在该怎么选

如果只是学习 Node 基础，`CommonJS` 往往更容易入门：

```js
const fs = require("fs");
module.exports = { a, b };
```

如果是现代项目，通常更推荐 `ES Module`：

```js
import fs from "fs";
export { a, b };
```

可以先这样记：

- `CommonJS` 更容易上手
- `ES Module` 更现代、更标准

## 10. 一句话总结

- `import` 和 `require` 的区别，不是 `TS` 和 `JS` 的区别
- 本质上是 `ES Module` 和 `CommonJS` 的区别
- `module.exports / require` 更直白
- `export / import` 更标准、更现代
